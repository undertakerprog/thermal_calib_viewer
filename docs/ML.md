# Модель контроля качества (ML)

Документ описывает обучение и применение ML‑модели для автоматического определения
“плохих” кадров в калибровочных RAW‑наборах. Модель **не генерирует** кадры —
она лишь оценивает качество и помогает найти первую плохую температуру.

## 1. Задача
Вход: серия RAW‑кадров, отсортированных по температуре.  
Выход: оценка качества каждого кадра (good/bad) и температура начала дефектов.

Сценарий использования:
1) Мы генерируем `gen` папку по алгоритму.
2) ML‑модель проверяет результат и сообщает, есть ли дефекты.

## 2. Разметка данных
Источник разметки — пары папок:

```
data-ml/
  good/  <- идеальные папки (gen)
  bad/   <- соответствующие “сырые” папки
```

Связь пар по имени: `good/<name>(gen)` ↔ `bad/<name>`.

Разметка кадра:
- **good** — температура присутствует в `good` папке.
- **bad** — температура есть только в `bad` папке.

## 3. Признаки (features)
Мы используем только **табличные признаки**, поэтому модель быстая и стабильная:

### 3.1. Diff‑метрики (из тройки кадров)
Для каждой тройки строится `diff`:
```
calib = interp(prev, next)   (Q15, shift=15)
diff  = (curr - min(curr)) - calib
```

Из `diff` берутся:
- `diff_std` — общий уровень шума
- `diff_stripe` — вертикальные полосы (колонковая дисперсия)
- `diff_vignette` — виньетка (амплитуда радиального профиля)
- `diff_mean_abs` — среднее |diff|
- `diff_p95` — 95‑й перцентиль |diff|

### 3.2. Raw‑метрики (по текущему кадру)
- `raw_mean`
- `raw_std`
- `raw_p1`, `raw_p99`

## 4. Подготовка датасета
Скрипт: `ml/ml_prepare.py`  
Выход: `data-ml/dataset.csv`

```
python ml/ml_prepare.py
```

Настройки:
- `--downsample` и `--metrics-downsample` уменьшают размер данных, ускоряя работу.

## 5. Обучение модели
Скрипт: `ml/ml_train.py`  
Модель: `RandomForest` (быстро, без GPU).

```
python ml/ml_train.py
```

Вывод:
```
data-ml/model.pkl
```

### Почему RandomForest
- Устойчив к шумам.
- Хорошо работает на табличных метриках.
- Быстрый инференс.

## 6. Инференс / проверка папки
Скрипт: `ml/ml_predict.py`

```
python ml/ml_predict.py --folder "путь_к_raw_папке"
```

Параметры:
- `--threshold` (по умолчанию 0.5)
- `--start-temp` (по умолчанию 50°C)

Вывод:
```
ML: bad from 55.40 (prob 0.83)
```

## 7. Интеграция в GUI
В `viewer_modes.py` добавлена кнопка **ML Check**:
- загружает `data-ml/model.pkl`
- считает признаки
- выводит температуру первого bad

## 8. Рекомендации
- Для высокой точности важнее **не пропускать bad** → лучше снижать threshold.
- После появления новых “боевых” данных модель можно дообучить,
  просто пере‑запустив `ml_prepare.py` и `ml_train.py`.
