# Математическое описание

Ниже приведено подробное описание математической части проекта: обработка RAW, построение
StdCalib‑diff, метрики качества, детекция “первого плохого” и генерация новых кадров.

## 1) Нотация
- Температуры: `T_i`.
- Калибровочные кадры: `F_i(x, y)` — значение пикселя в `int16`.
- Тройка температур: `T_{i-1}, T_i, T_{i+1}`.
- Индексы пикселей: `(x, y)`, где `x ∈ [0, W-1]`, `y ∈ [0, H-1]`.

## 2) Интерполяция “калибровки” между соседями
Для тройки `(prev, curr, next)` вычисляется интерполированная калибровка:

```
alpha = (T_i - T_{i-1}) / (T_{i+1} - T_{i-1})
beta  = 1 - alpha
Calib(x, y) = alpha * F_{i-1}(x, y) + beta * F_{i+1}(x, y)
```

В коде используется fixed‑point арифметика (Q15):
```
w0 = alpha * 2^15
w1 = beta  * 2^15
Calib = (F_{i-1} * w0 + F_{i+1} * w1 + 2^(15/2)) >> 15
```
Так сохраняется точность без переполнения.

## 3) StdCalib‑diff (основа для анализа качества)
Для текущего кадра формируется отклонение от интерполированной калибровки:

```
F_norm(x, y) = F_i(x, y) - min(F_i)
Diff(x, y) = F_norm(x, y) - Calib(x, y)
```

Именно `Diff` отражает неравномерности, виньетку и полосы.

## 4) LHE (Local Histogram Equalization)
Для визуализации Diff применяется локальная гистограммная коррекция:
1) строится гистограмма `H` размера `≈ 1.5 * int16_max`;
2) клиппинг по `clipLo/clipHi`, затем `clipLimit`;
3) строится LUT и происходит маппинг в `uint8`.

Это нужно для **визуальной диагностики**, не для метрик.

## 5) Метрики качества
Метрики вычисляются **по Diff** (до LHE), чтобы измерения были физически значимы.

### 5.1) Общий уровень шума (std)
```
std = sqrt( (1/N) * Σ (Diff - mean(Diff))^2 )
```

### 5.2) Полосы (vertical stripes)
1) Среднее по колонкам:
```
C(x) = mean_y Diff(x, y)
```
2) Сглаживание (скользящее среднее):
```
S(x) = mean_{k ∈ window} C(x + k)
```
3) Метрика:
```
stripe = std( C(x) - S(x) )
```

### 5.3) Виньетка (radial profile)
1) Радиальный профиль:
```
P(r) = mean_{(x,y): dist((x,y), center)=r} Diff(x, y)
```
2) Метрика:
```
vignette = max(P) - min(P)
```

## 6) Робастная нормализация (median/MAD)
Для каждой метрики строится робастный z‑score:

```
median = median(metric)
MAD = median(|metric - median|)
z = (metric - median) / (MAD * 1.4826)
```

Используется нижняя часть распределения (примерно 30%), чтобы “плохие” кадры не влияли на базу.

## 7) Итоговый score и детекция “первого плохого”
Итоговый score — взвешенная сумма положительных z‑score:

```
score = w_std * max(z_std, 0)
      + w_stripe * max(z_stripe, 0)
      + w_vignette * max(z_vignette, 0)
```

Далее:
1) находим “устойчивое превышение” после `T >= 50°C` (streak);
2) сдвигаем начало влево к первым “ранним” признакам;
3) гарантируем отсутствие “опоздания” (пропущенных плохих).

## 8) Генерация новых кадров (polynomial)
Цель: заменить все кадры, начиная с плохого.

### 8.1) Сетка температур
```
T_gen = anchor, anchor + step, ..., 70.0
step = 1.25°C
```

`anchor` = последний нормальный кадр.

### 8.2) Полиномиальная аппроксимация по каждому пикселю
Для каждого пикселя `(x, y)`:
```
F(x,y,T) ≈ Σ_{k=0..N} a_k(x,y) * T^k
```
Коэффициенты `a_k` находятся по МНК (least squares) на “хороших” кадрах.

Затем:
```
F_gen(x,y,T_gen) = polyval(a_k, T_gen)
```

### 8.3) Сохранение
Формат `FpaImage`:
```
float tAdc
int cols
int rows
int type (3)
uint64 data_size
int16 data
```

## 9) Причины выбора степени полинома
`N=3` даёт более устойчивую аппроксимацию и меньше “звон” на краях,
чем `N=5`. Это критично при вырезании части кадров перед генерацией.
