# Thermal calib viewer (modes)

Этот проект сфокусирован на просмотре RAW‑наборов калибровки и диагностике качества.
Ниже описана только логика обработки и отображения (GUI специально не расписывается).

## Общий пайплайн (для всех режимов)
- Файлы ищутся по имени `calib_<temp>_<adc>.raw` (лишние подчёркивания допускаются).
- `<temp>` парсится как десятичная температура; сортировка по `(temp, adc, path)`.
- Строятся перекрывающиеся тройки: для индекса `i` используются `Ti`, `T(i+1)`, `T(i+2)`.
- Плитка строится только если существуют все три кадра; последние две температуры плиток не дают.
- Все режимы отдают на вход рендера 8‑битный grayscale `uint8`.

## Режим: Raw (per‑frame scale) [raw_local]
Цель: показать каждый кадр максимально контрастно, независимо от остальных.

Шаги:
1) Читаем `Ti` как `uint16` (учитываем возможный заголовок 24 байта).
2) Считаем перцентили `p1` и `p99` только по этому кадру.
3) Клипаем в диапазон `[p1, p99]` и линейно масштабируем в `[0..255]`.
4) Рендерим 8‑битное изображение.

Эффект: локальные дефекты видны хорошо, но яркость между температурами не сравнима.

## Режим: Raw (global scale) [raw_global]
Цель: показывать все кадры в едином масштабе, чтобы сравнивать уровни между температурами.

Шаги:
1) Читаем `Ti` как `uint16`.
2) Строим глобальную шкалу по выбранным кадрам:
   - Из каждого кадра берём подвыборку (каждый 8‑й пиксель).
   - Объединяем выборки и считаем `p1`, `p99`.
3) Клипаем `Ti` в `[p1, p99]`, масштабируем в `[0..255]`.
4) Рендерим 8‑битное изображение.

Эффект: хорошо видны тренды яркости по температуре, но контраст может быть ниже.

## Режим: Diff from triple mean [diff_mean]
Цель: подсветить отклонения `Ti` от ближайшего температурного окружения.

Шаги:
1) Читаем `Ti`, `T(i+1)`, `T(i+2)` как `uint16`.
2) Вычисляем среднее по трем кадрам: `(Ti + T(i+1) + T(i+2)) / 3`.
3) Считаем разность: `Ti - mean` (знаковая).
4) Находим `p1`/`p99` разности.
5) Масштабируем так, чтобы `0` стал серым (128), отрицательные — темнее, положительные — светлее.

Эффект: локальные аномалии становятся заметнее, глобальные тренды подавляются.

## Режим: StdCalib (LHE) [calib_interp]
Цель: воспроизвести “стандартный тест калибровки” из референсной программы.

Шаги:
1) Читаем `Ti-1`, `Ti`, `Ti+1` как `int16` (с учётом 24‑байтового заголовка).
2) Считаем вес интерполяции:
   - `prev_weight = (Ti - Ti-1) / (Ti+1 - Ti-1)`; если знаменатель 0, берём 0.5.
3) Строим интерполированную “калибровку” фиксированной точности:
   - `calib = prev * w0 + next * w1`, веса в формате Q15 (`shift=15`).
4) Нормализуем текущий кадр:
   - `curr = curr - min(curr)`.
5) Вычитаем калибровку:
   - `diff = curr - calib`.
6) Применяем LHE (локальное гистограммное выравнивание):
   - Размер гистограммы: `int16_max * 1.5`.
   - Параметры клиппинга: `clipLo=100`, `clipHi=100`, `clipLimit=100`.
   - Порог масштабирования диапазона: `rangeScaleKeyPoint=175`.
   - Строим LUT и отображаем в `uint8`.

Эффект: хорошо проявляет виньетку, полосы и неравномерности, как в оригинальном тесте.

## План расширения: автоматическая генерация и контроль качества
Ниже — будущие шаги, которые будут добавляться в проект. Это не GUI‑описание, а инженерное
описание алгоритмов и мест интеграции.

### 1) Автоматическая генерация наборов (gen)
Цель: получить новые калибровочные кадры для заданного диапазона температур.

Базовый алгоритм (как в `generateCalibs` из референса):
1) Имеем набор исходных калибровок `{(Ti, Fi)}`, где `Fi(x, y)` — кадр `int16`.
2) Для каждой фиксированной координаты `(x, y)` аппроксимируем зависимость
   `Fi(x, y)` от `Ti` полиномом степени `N`:
   - находим коэффициенты `p` по МНК;
   - вычисляем значения на сетке температур `Tg`.
3) Формируем новые кадры `Fg(x, y) = polyval(p, Tg)`.

На старте допускается более простой путь:
- линейная интерполяция между ближайшими температурами.

### 2) Метрики качества (без ML)
Цель: формально измерить “ровность” и наличие артефактов.

Предлагаемый набор метрик:
1) **Неравномерность яркости**:
   - `mu = mean(I)`, `sigma = std(I)`;
   - относительная неравномерность: `sigma / max(mu, eps)`.
2) **Радиальная виньетка**:
   - строим радиальный профиль `P(r)` (среднее по кольцам);
   - оцениваем амплитуду: `max(P) - min(P)`.
3) **Полосы (вертикальные/горизонтальные)**:
   - FFT по строкам/столбцам;
   - выделение пиков в спектре.
4) **Шум**:
   - разность `I - blur(I)` и RMS остатка.

Результат: набор чисел, по которым можно ставить порог “кадр плохой/хороший”.

### 3) Модель контроля качества (ML‑фильтр)
Цель: заменить ручные пороги на статистическую модель.

Подход:
1) Собираем датасет кадров с ручной разметкой (good/bad).
2) Используем признаки из пункта 2 как вектор `x`.
3) Обучаем классификатор (RandomForest/LightGBM/SVM).
4) Модель возвращает класс и уверенность.

Важно: модель **не генерирует** калибровки. Она только оценивает качество.

### 4) Интеграция в пайплайн
Ожидаемый поток:
1) Загрузка набора RAW.
2) Генерация `gen` (интерполяция/полином).
3) Прогон метрик → (опционально) модель.
4) Отображение результатов в viewer и сохранение итоговых наборов.

### 5) Конфигурация и воспроизводимость
Рекомендуется хранить:
- параметры генерации (`from`, `to`, `step`, `polyN`);
- параметры метрик и пороги;
- версию модели и параметры обучения.
